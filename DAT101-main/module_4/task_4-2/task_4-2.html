1. What is an array? A short and concise answer is expected here!

    2. An array is like a collection of stuff (values or objects) you want to keep together in one place, indexed by numbers starting at 0. Think of it as a row of lockers for your data basically.

  2. What does it mean to "hardcode" when we talk about programming?

    3. Hardcoding is when you write fixed values directly into your code, like writing "5" instead of making it a variable. It's cool for quick stuff but not great for flexibility later as things will become quite tedious.

  3. Explain in plain text what happens in task 3, and how we can traverse (iterate) the array when we (humans) don't "know" how long the array is.

    4. In task 3, we split a sentence into words using `.split(" ")`, creating an array. Then, we use `forEach` to go through each word, printing it with its index. If we don't know the array length, we use loops like `for`, `forEach`, or even `while`, which handle unknown lengths for us.

  4. Why is it beneficial to store instances of a class in arrays, as in task 4?

    5. Putting class instances in an array makes it easier to handle a bunch of similar objects at once. You can loop through them, filter them, or apply methods without needing separate variables for each one.

  5. Try your best to explain the EWeekDays object from task 7. How do we use the "value" attribute, for example? In what way do "WeekDay", "Workdays", and "Weekends" differ from each other? If we wanted to create a new element "WorkoutDays", which would contain "Monday", "Wednesday", and "Thursday", how would it ideally look?

    6. The `EWeekDays` object groups days with their names and numeric values. The "value" is like a unique code (often used for grouping or some stuff). "WeekDay" entries are single days, "Workdays" combine multiple days into one group, and "Weekends" do the same for Saturday and Sunday. A "WorkoutDays" element would look like this:

    WorkoutDays: { value: 0x01 + 0x04 + 0x08, name: "WorkoutDays" }

  6. In task 8, we will use a "callback" routine when sorting. Explain how such a "callback" routine works.

    7. A callback is a function you pass into another function (like `sort`). In sorting, it takes two items, compares them, and tells the sorting function how to order them (e.g., ascending or descending).

  7. What happens in the sorting routines you have created? Try to explain how the array can figure out if an element should come before or after another. The best way to answer this is to actually print both parameters each time the sorting routine runs.

    8. The sorting function compares two items at a time (`a` and `b`) using the callback. If it returns a negative number, `a` comes before `b`. Positive means `b` comes before `a`, and 0 means they're equal.

  8. In task 10, we will create an array with rows and columns. Explain what the first element of the array contains.

    9. The first element of the array contains the first row, which itself is an array of elements for each column. So, if it's a 5x9 grid, the first element looks something like..: ["Row 0, Col 0", "Row 0, Col 1", ..., "Row 0, Col 8"].
