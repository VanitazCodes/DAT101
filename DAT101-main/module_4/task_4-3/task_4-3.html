  1. In this assignment, we use the DOM. Explain how this library can help us develop interactive web pages.

    2. The DOM is like a control panel for your webpage. It lets you mess around with HTML elements using JavaScript, so you can make your page way cooler and more interactive. You can update text, add animations, or even create elements out of thin air. Basically, it's what turns a boring static page into something fun and alive.

  2. What is an event? Simply explain what happens when we add an event listener to an object.

    3. An event is just something that happens, like clicking a button, typing in a field, or even just moving your mouse. When you add an event listener, it's like saying, “Hey, JavaScript, keep an eye out for this thing happening.” When the event actually happens, it triggers a function, so you can respond to it—like showing an alert, updating some text, or anything else you wanna do.

  3. Sometimes we need to use an event object to get data from the event. Explain how this event object is structured, don't explain all functions or attributes. Try to explain, for example, how to retrieve data from a keyboard or mouse.

    4. The event object is like a cheat sheet that tells you everything about what just happened. For example:

    • With a keyboard event, `event.key` tells you which key was pressed (like “Enter” or “A”).
    • With a mouse event, you can use `event.clientX` and `event.clientY` to find out where the click happened on the screen.
    It's super handy because you don't have to guess what the user did—the event object has all the details you need.

  4. Explain the main difference between DOM id and DOM name. Can we have identical IDs?

    5. IDs and names are kinda similar but do different things:

    • "IDs" are like social security numbers—totally unique and only one element can have a specific ID on a page.
    • "Names" are more like tags you can give to a group of elements, like form inputs, so they're easier to handle together.
    If you try to use the same ID twice, you'll confuse JavaScript, and things might break. So yeah, keep IDs unique!
