1. What is the difference between invoking and creating a function?

    2. Creating a function is when you write it out, basically setting it up with what it should do. Invoking a function is when you actually use it to do its job.

  2. Explain the syntax regarding semicolons and code blocks.

    3. In JavaScript, semicolons end a line of code (but they're sometimes optional). Code blocks are the parts wrapped in `{ }` brackets, and they hold all the instructions for things like functions and loops.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. If a function doesn't return anything, trying to store its result will just give you `undefined`, which means there's no value.

  4. How do we separate parameters from each other when creating a function?

    5. We separate parameters by putting commas between them inside the parentheses when writing the function. For example: `function example(param1, param2) { }`.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. When you create a function, parameter names are placeholders. When you use (or "invoke") the function, these placeholders get replaced with the actual values you pass in.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. If a function call is missing a parameter, JavaScript just leaves that spot as `undefined` because no value was given for it.

  7. Explain what happens when we use recursive functions.

    8. A recursive function is a function that calls itself. Each time it calls itself, it gets closer to a stopping point, which is called the base case. If there's no base case or if it never gets there, it could keep going forever and crash.
